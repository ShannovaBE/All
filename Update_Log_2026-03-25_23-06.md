## 1. Executive Summary

This document outlines the critical bug fixes, infrastructure optimizations, and major feature additions applied to the Shannova Platform since the initial deployment.

## 2. Infrastructure & Cost Optimization Fixes

### A. The Server Cold-Start Fix (Pre-Baking AI)
*   **The Problem:** When uploading a file, the `curl` request (and browser) timed out with an `HTTP2 framing layer` error. This happened because Google Cloud Run spun up an empty container and spent 3 minutes attempting to download 1.5GB of Hugging Face models (`DeBERTa` and `i2b2`) over the network before it could process the file.
*   **The Fix:** We injected a `download_models.py` script directly into the backend `Dockerfile`. Now, the models are downloaded *during* the build phase and "baked" permanently into the hard drive of the Docker image. 
*   **Result:** Uploads are now processed instantly without network download latency.

### B. OOM (Out of Memory) Crash Fix
*   **The Problem:** The backend was crashing because allocating Pandas Dataframes alongside two massive PyTorch AI models exceeded the default 2GB RAM limit on Cloud Run.
*   **The Fix:** Increased the maximum memory allocation for the backend container to `4Gi` (4 Gigabytes).

---

## 3. Data Pipeline Analytics Upgrades

The backend `pipeline.py` was fundamentally rewritten to extract deep analytics from uploaded datasets before they reach the database.

### A. Information Theory Metrics (Alpha Engine)
- The pipeline now uses `scipy.stats.entropy` to calculate the **Shannon Entropy** across categorical columns. This gives buyers a mathematical representation of the dataset's variance and unpredictability (i.e., higher entropy means richer information).
- It also mathematically calculates and stores the Exact Row Count, Column Count, and Missing Data Percentages.

### B. The Privacy Report (Gamma Engine)
- The Gamma PII Eraser now generates a quantitative report.
- It compares the original DataFrame against the scrubbed DataFrame to count exactly how many cells contained sensitive information.
- It returns a compliance string (e.g., *"Scrubbed 14 cells containing sensitive PII"*).

## 4. Frontend UI/UX Improvements

The Next.js React application was updated to visually display the new pipeline analytics and improve user navigation.

### A. Navigation Renaming
- The ambiguous dropdown options under "Marketplace" were changed.
- "Upload" was renamed to **Selling** (the entry point for data providers).
- "Selling" was renamed to **Buying** (the entry point for data consumers).

### B. The Dataset Details Page
The `/selling/[hash]/page.tsx` was heavily updated to include three new UI components:
1.  **🛡️ Privacy & PII Report:** Highlights the Gamma engine's redaction count in red or green.
2.  **📊 Dataset Statistics:** A stylized grid displaying the Row Count, Missing Data percentage, and Shannon Entropy.
3.  **🔎 Dataset Sample Preview:** Instead of buying blind, users now see a visually styled HTML table rendering the first 5 rows of the anonymized CSV data.

*Note: All Frontend UI additions were rigorously typed to resolve Next.js 15 `unknown` ReactNode compilation errors.*
